# 股票量化分析工具 — 技术规划文档

## 1. 项目概述

### 1.1 项目定位
面向A股市场的量化分析Web工具，一期聚焦**技术指标计算与可视化**、**多因子筛选**两大能力。支持11项常用技术指标，提供可视化规则构建器进行全市场扫描筛选。

### 1.2 核心目标

| 目标 | 描述 | 现状 |
|------|------|------|
| 技术指标计算 | 计算MA/EMA/MACD/RSI等11项指标，输出信号 | 已实现 |
| 交互式图表 | K线图叠加均线/布林带，多子图展示MACD/RSI/KDJ等 | 已实现 |
| 多因子筛选 | 可视化AND/OR规则构建器，全市场5529只A股扫描 | 已实现 |
| 策略预设 | 趋势追踪/超跌反弹/放量突破 三种内置策略 | 已实现 |
| 数据缓存 | 日线缓存24h、股票列表缓存1h，避免重复API调用 | 已实现 |

## 4. 核心模块设计

### 4.1 数据获取层 (`server/src/services/fetcher.ts`)

**设计原则：** 封装东方财富公开API，统一返回格式，与上游API解耦。

**关键实现：**
- **K线获取**: `fetchKline(code, startDate, endDate, period, adjust)` → 返回 `KlineData[]`
  - 自动识别沪深市场生成 `secid`（沪市 `1.xxx`，深市 `0.xxx`）
  - 支持4种周期：`60min`（60分钟线）、`daily`（日线）、`weekly`（周线）、`monthly`（月线）
  - 60分钟线单次返回上限800条（`lmt=800`），其余周期500条
  - 返回标准化字段：`date, open, close, high, low, volume, amount`
  - 前复权(qfq)为默认复权方式

- **股票列表获取**: `fetchStockList()` → 返回 `StockInfo[]`
  - 市场过滤器：`m:0+t:6(深A), m:0+t:80(创业板), m:1+t:2(沪A), m:1+t:23(科创板)`
  - **分页策略**：API每页上限100条，5529只股票需56页，每页间隔100ms
  - 含重试机制：单页失败后延时500ms重试一次
  - 首页加载耗时约6秒，结果缓存1小时

- **实时行情**: `fetchRealTimeQuote(symbols[])` → 批量获取

### 4.2 缓存层 (`server/src/services/cache.ts`)

**持久化策略：** 缓存数据同时写入内存和磁盘（`server/data/*.json`），服务器重启后自动加载未过期的缓存，避免反复请求东方财富API触发限流。

**关键特性：**
- **磁盘持久化**：每个缓存key对应一个JSON文件，包含数据+过期时间
- **自动恢复**：启动时扫描 `data/` 目录，加载未过期缓存，清除过期文件
- **TTL机制**：日线数据24h，股票列表1h，实时行情不缓存
- **并发去重**：`pending` Map 防止同一key的并发重复请求
- **空值检测**：列表路由中验证数据非空后再缓存，防止缓存空结果

**缓存生命周期：**
```
请求 → 内存命中? → 磁盘恢复? → API获取 → 写内存+写磁盘
         │            │
         └─ 返回 ─────┘
                        过期 → 下次请求时删除文件+重新获取
```

### 4.3 指标计算层 (`server/src/indicators/`)

**设计模式：** 接口抽象 + 注册表工厂

```typescript
// 接口定义
interface Indicator {
  name: string;
  category: 'trend' | 'oscillator' | 'volume' | 'volatility';
  params: Record<string, number>;
  compute(data: KlineData[]): IndicatorResult;
}

// 工厂模式创建
const ind = getIndicator("RSI", { period: 12 });
ind.compute(klineData);
```

**已实现的11个指标：**

| 分类 | 指标 | 输出列 | 信号类型 |
|------|------|--------|----------|
| 趋势 | MA | MA_{period} | 金叉/死叉 |
| 趋势 | EMA | EMA_{period} | 金叉/死叉 |
| 趋势 | MACD | DIF, DEA, BAR | DIF上穿/下穿DEA |
| 趋势 | BOLL | MID, UPPER, LOWER | 突破上轨/下轨 |
| 震荡 | RSI | RSI_{period} | >70超买 / <30超卖 |
| 震荡 | KDJ | K, D, J | K上穿/下穿D, 超买超卖 |
| 震荡 | WR | WR_{period} | <20超买 / >80超卖 |
| 量能 | VOL | VOL_MA_{period} | 放量>2倍均量 |
| 量能 | OBV | OBV, OBV_MA | OBV上穿/下穿均线 |
| 量能 | VolumeRatio | VR | >1.5放量 / >3极放 |
| 波动 | ATR | ATR_{period} | 波动加剧(30%+) |

**交叉检测公共方法：**
- `crossAbove(fast[], slow[])` → 快线上穿慢线
- `crossBelow(fast[], slow[])` → 快线下穿慢线
- `sma(values[], period)` → 简单移动平均
- `ema(values[], period)` → 指数移动平均

### 4.4 筛选引擎层 (`server/src/screening/`)

**规则模型：**
```typescript
RuleCondition {
  indicator: "RSI"       // 指标名
  operator: ">"          // > | < | >= | <= | == | cross_above | cross_below
  value: 25              // 阈值
  params: { period: 12 } // 指标参数
}

RuleGroup {
  logic: "AND" | "OR"    // 逻辑关系
  conditions: [...]       // 支持嵌套分组
}
```

**筛选流程：**
1. `validateRules()` — 校验规则合法性（指标是否存在、参数是否有效）
2. `evaluate(data, rules)` — 对单只股票的K线数据评估规则
3. `ScreeningEngine.screen()` — 遍历全市场，每50只yield事件循环
4. 结果按涨跌幅降序排列

**内置3个预设策略：**

| 策略 | 规则组合 | 适用场景 |
|------|----------|----------|
| 趋势追踪 | EMA5>EMA20 AND MACD金叉 AND VOL>1.2倍均量 | 追涨，趋势延续 |
| 超跌反弹 | RSI(12)<25 AND OBV上穿均线 AND WR(14)>80 | 抄底，超卖反转 |
| 放量突破 | 价格>BOLL上轨 AND 量比>2 | 突破行情 |

## 7. 关键设计决策

### 7.1 为何选择东方财富API而非akshare
- akshare 是 Python 库，本项目使用 Node.js 技术栈
- 东方财富公开API是HTTP接口，语言无关，直接调用
- 无需申请token，零配置即可使用
- 覆盖全A股5529只股票，数据完整

### 7.2 为何分页获取股票列表
- 东方财富API单页上限100条（pz参数被忽略）
- 全量5529只需要56次请求，总耗时约6秒
- 缓存1小时后直接命中，不再重复请求

### 7.3 为何禁用Express ETag
- Express默认对GET请求生成ETag
- 首次请求返回空数据后，浏览器缓存空响应ETag
- 后续请求发送If-None-Match → 服务端返回304 → 浏览器使用缓存的空数据
- 对于动态数据API，禁用ETag更可靠

### 7.4 为何指标计算放在后端
- 计算逻辑与前端解耦，未来可复用于其他客户端
- JSON传输计算结果，前端专注渲染
- TypeScript后端可直接复用指标计算逻辑做筛选

## 10. 已知限制与注意事项

| 限制 | 说明 | 影响 |
|------|------|------|
| API限流 | 东方财富API对高频请求可能限流 | 首屏加载需6秒（获取股票列表） |
| 数据精度 | 东方财富日线数据为前复权价格 | 适合技术分析，不适合绝对价格回测 |
| 实时性 | 无WebSocket推送，依赖轮询 | 不适合日内高频交易 |
| 浏览器内存 | Plotly渲染大量数据可能卡顿 | 建议数据范围不超过5年日线 |
| 筛选性能 | 全市场扫描需逐个计算指标 | 5529只股票约需1-2分钟 |
